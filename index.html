System Architecture

High-level system design showing component interactions and data flow

Component Architecture

The system consists of four main layers: input processing, NEAT engine, game simulation, and fitness evaluation. The NEAT engine manages population evolution while the game environment provides the fitness landscape.

Key Components

  • Population Manager: Handles 20 genomes per generation
  • Neural Networks: Feedforward networks created from genomes
  • Game Environment: Physics simulation with collision detection
  • Fitness Evaluator: Multi-component reward system
  • Species Handler: Automatic grouping based on compatibility

Data Flow Architecture

Data flows from genome initialization through neural network creation, game simulation, fitness evaluation, and back to population evolution in a continuous cycle.

Data Processing Stages

  • Input: Game state (bird position, pipe locations, velocity)
  • Processing: Neural network forward pass with tanh activation
  • Output: Jump decision (threshold-based)
  • Feedback: Fitness scores drive evolutionary selection

Genetic Lifecycle

Complete NEAT evolutionary process from population initialization to successful convergence

1
Population Initialization
Start with 20 random genomes. Each genome contains connection genes with random weights and minimal structure. Initial networks have 4 inputs connected directly to 1 output neuron.
2
Neural Network Creation
Convert each genome into a functional feedforward neural network. Networks process normalized game inputs through tanh activation functions to produce jump decisions.
3
Game Simulation
Each neural network controls one bird in the game environment. Birds face physics (gravity, velocity) and obstacles (pipes) requiring precise timing for successful navigation.
4
Fitness Evaluation
Multi-component fitness calculation: +0.1 survival bonus, +0.05 centering reward, +0.02 progress incentive, +15 pipe passage achievement, penalties for collisions.
5
Species Formation
Group similar genomes using compatibility distance (threshold: 3.5). Species protect innovation and maintain genetic diversity through separate reproduction pools.
6
Selection & Reproduction
Select top 25% performers from each species. Apply elitism (preserve 2 best genomes). Create offspring through crossover and mutation to maintain population size.
7
Mutation Operations
Weight mutations (60%): Gaussian perturbation. Bias mutations (40%): Node threshold adjustment. Structural mutations (5-20%): Add/remove connections and nodes.
8
Termination Check
Evaluate if best fitness exceeds threshold (300). If successful, return winning genome. Otherwise, continue evolution cycle with new generation.

Feedforward Neural Network

Detailed explanation of the 4-input neural network architecture and decision-making process

Network Architecture

The neural network uses a simple but effective 4-input to 1-output feedforward architecture. This minimal structure allows for fast evolution while providing sufficient complexity for the task.

Input Processing

Four normalized game state variables feed into the network:

  • Bird Y Position: bird.y / WINDOW_HEIGHT (range: 0-1)
  • Gap Distance: (bird.y - pipe_center) / (WINDOW_HEIGHT/2) (range: -1 to 1)
  • Velocity: bird.velocity / 20.0 (range: approximately -1 to 1)
  • Pipe Distance: (pipe.x - bird.x) / WINDOW_WIDTH (range: 0-1)

Mathematical Operations

# Forward Pass Calculation
z = Σ(wᵢ × xᵢ) + bias
output = tanh(z)

# Decision Threshold
if output > 0.3:
    bird.jump()
else:
    # Continue falling

Activation Function

The hyperbolic tangent (tanh) function provides bounded output (-1 to 1) and smooth gradients for stable learning. The 0.3 threshold creates a bias toward jumping when confidence is moderate to high.
